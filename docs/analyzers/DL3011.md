# DL3011 - Simplify conditional assignment to ternary

Detects `if` statements that assign different values to the same variable in both branches and suggests simplifying to a single ternary assignment.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3011 | Style    | Warning          | Yes      |

## Why

An `if` statement that assigns one value in the `if` branch and another in the `else` branch to the same target is equivalent to a single ternary assignment. Simplifying removes unnecessary control flow and makes the intent clearer.

## When the diagnostic fires

- `if (cond) x = a; else x = b;` -- inline bodies
- `if (cond) { x = a; } else { x = b; }` -- block bodies
- Method call values: `if (cond) x = Foo(); else x = Bar();`
- Short expressions: `if (cond) x = a + 1; else x = b + 1;`
- Assignment to properties: `if (cond) obj.Value = a; else obj.Value = b;`

## When the diagnostic does NOT fire

- No else clause (`if (cond) x = a;` alone)
- Both values are boolean literals (`if (cond) x = true; else x = false;`) -- that's DL3009's domain
- Different assignment targets (`if (cond) x = a; else y = b;`)
- Nested ternary in value (`if (cond) x = a ? 1 : 2; else x = 3;`)
- Either value text longer than 60 characters
- Multiple statements in a branch
- Inside `Expression<Func<>>` lambdas (expression trees)
- Compound assignment operators (`+=`, `-=`, etc.)

## Examples

### Violation

```csharp
if (cond) x = 1; else x = 2;                   // DL3011
if (cond) { x = 1; } else { x = 2; }           // DL3011
if (cond) x = Foo(); else x = Bar();            // DL3011
if (cond) obj.Value = a; else obj.Value = b;    // DL3011
```

### Fixed

```csharp
x = cond ? 1 : 2;
x = cond ? 1 : 2;
x = cond ? Foo() : Bar();
obj.Value = cond ? a : b;
```

### No diagnostic

```csharp
// No else clause
if (cond) x = 1;

// Boolean literals (DL3009)
if (cond) x = true; else x = false;

// Different targets
if (cond) x = 1; else y = 2;

// Nested ternary
if (cond) x = y ? 1 : 2; else x = 3;

// Long value
if (cond) x = "This is a very long string value that exceeds sixty characters limit here"; else x = "short";

// Compound assignment
if (cond) x += 1; else x += 2;
```

## Code fix

Replaces the entire `if`/`else` construct with `target = condition ? trueValue : falseValue;`. Preserves leading trivia from the original `if` statement.
