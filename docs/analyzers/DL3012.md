# DL3012 - Merge nested if statements

Detects nested `if` statements that can be merged into a single `if` with a combined `&&` condition.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3012 | Style    | Warning          | Yes      |

## Why

Nested `if` statements without else clauses are equivalent to a single `if` with the conditions combined using `&&`. Merging them reduces indentation and makes the combined guard condition explicit.

## When the diagnostic fires

- `if (a) { if (b) { body } }` -- inner block contains only another if
- `if (a && c) { if (b) { body } }` -- outer condition is already `&&`
- Deeply nested: `if (a) { if (b) { if (c) { body } } }` -- flags the outermost pair only

## When the diagnostic does NOT fire

- Outer if has an else clause
- Inner if has an else clause
- Outer block contains other statements (before or after the inner if)
- The single statement inside the outer block is not an if statement
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
if (a)                   // DL3012
{
    if (b)
    {
        DoStuff();
    }
}
```

### Fixed

```csharp
if (a && b)
{
    DoStuff();
}
```

### Parenthesisation

When either condition is a logical OR expression, the code fix wraps it in parentheses to preserve semantics:

```csharp
// Before
if (a || b) { if (c) { DoStuff(); } }

// After
if ((a || b) && c) { DoStuff(); }
```

### No diagnostic

```csharp
// Outer if has else
if (a) { if (b) { DoStuff(); } } else { DoOther(); }

// Inner if has else
if (a) { if (b) { DoStuff(); } else { DoOther(); } }

// Other statements in block
if (a) { DoStuff(); if (b) { DoOther(); } }
```

## Code fix

Replaces `if (outer) { if (inner) { body } }` with `if (outer && inner) { body }`. Wraps either condition in parentheses when it is a logical OR or other low-precedence binary expression. Preserves leading trivia from the outer `if` statement and the inner body exactly.
