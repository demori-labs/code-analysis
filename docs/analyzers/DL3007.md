# DL3007 - Use declaration pattern instead of 'as' with null check

Detects `var t = x as T` followed by a null check `if (t != null)` or `if (t is not null)`, which should use a declaration pattern `if (x is T t)` instead.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3007 | Style    | Warning          | Yes      |

## Why

The `as` + null check pattern requires a separate variable declaration and a null check on the next line. A declaration pattern `if (x is T t)` combines the type check, cast, and null check into a single expression, reducing boilerplate and scoping the variable to where it's actually non-null.

## When the diagnostic fires

- `var t = x as T;` immediately followed by `if (t != null)` or `if (t is not null)`
- The variable is only used inside the if-body (not before, after, or in the else branch)

## When the diagnostic does NOT fire

- Variable is used between the declaration and the null check
- Variable is used after the if-statement (would lose scope)
- Variable is used in the else branch
- The null check is not the immediately following statement
- There are other statements between the `as` and the `if`
- Inside `Expression<Func<>>` lambdas

## Examples

### Violation

```csharp
var animal = o as Animal;     // DL3007
if (animal is not null)
{
    Console.WriteLine(animal.Name);
}
```

### Fixed

```csharp
if (o is Animal animal)
{
    Console.WriteLine(animal.Name);
}
```

### No diagnostic

```csharp
// Already using declaration pattern
if (o is Animal animal)
{
    Console.WriteLine(animal.Name);
}

// Variable used after the if
var animal = o as Animal;
if (animal is not null)
{
    Console.WriteLine(animal.Name);
}
Console.WriteLine(animal); // used after if — can't convert
```

## Code fix

Removes the `var t = x as T;` declaration and replaces the if-condition with `x is T t`. The variable name and the if-body are preserved.
