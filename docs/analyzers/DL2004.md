# DL2004 - Method parameter should be [ReadOnly]

Suggests adding `[ReadOnly]` to all method parameters. The user should mark each parameter as `[ReadOnly]` to enforce immutability, or `[Mutable]` to explicitly opt out.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL2004 | Usage    | Info             | Yes      |

## Why

Method parameters are mutable by default in C#. Adding `[ReadOnly]` prevents accidental reassignment and makes the intent explicit. [DL2001](DL2001-DL2002.md) enforces the attribute — once added, any reassignment is an error. Adding `[Mutable]` explicitly opts out of the suggestion, signalling that mutation is intentional.

## When the diagnostic fires

- The method has a body (concrete, virtual, or override)
- The parameter does not already have `[ReadOnly]` or `[Mutable]`
- The parameter is not `ref`, `out`, or `in`
- The parameter name does not start with `_`

Applies to methods, constructors, local functions, and extension block receiver parameters.

## When the diagnostic does NOT fire

- The parameter already has `[ReadOnly]` or `[Mutable]`
- The parameter has `ref`, `out`, or `in` modifier
- The parameter name starts with `_` (convention for intentionally unused)
- Abstract methods, interface methods (no body)
- Event handler methods (`object sender, EventArgs e`)
- Entry point `static void Main(string[] args)`
- Extension block unnamed receiver (`extension(string)` — static extensions)

## Examples

### Violation

```csharp
public void Process(int id)         // DL2004
{
    Console.WriteLine(id);
}

public void Normalize(int id)       // DL2004 — fires even if reassigned
{
    id = Math.Abs(id);
    Console.WriteLine(id);
}

public C(string name)               // DL2004 — constructor parameter
{
    _name = name;
}

// Extension block receiver
public static class Extensions
{
    extension(string source)        // DL2004 on 'source'
    {
        public bool IsEmpty => source.Length == 0;
    }
}
```

### Fixed

```csharp
using DemoriLabs.CodeAnalysis.Attributes;

public void Process([ReadOnly] int id)
{
    Console.WriteLine(id);
}

public void Normalize([Mutable] int id)  // Opted out — mutation is intentional
{
    id = Math.Abs(id);
    Console.WriteLine(id);
}

public C([ReadOnly] string name)
{
    _name = name;
}
```

### No diagnostic

```csharp
// Suppressed with [Mutable]
public void Process([Mutable] int id) { }

// Already marked [ReadOnly]
public void Process([ReadOnly] int id) { }

// Abstract — no body
public abstract void Process(int id);

// Unnamed extension receiver — static extension
public static class Extensions
{
    extension(IEnumerable<int>)
    {
        public static IEnumerable<int> Empty => Enumerable.Empty<int>();
    }
}
```

## Code fix

**Add [ReadOnly] attribute** — adds `[ReadOnly]` to the parameter and inserts `using DemoriLabs.CodeAnalysis.Attributes;` if not already present.
