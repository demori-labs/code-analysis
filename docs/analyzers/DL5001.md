# DL5001 - Possible multiple enumeration

Detects when an `IEnumerable<T>` or `IEnumerable` is enumerated more than once in the same scope, which can cause performance issues or unexpected behaviour.

| Rule   | Category    | Default Severity | Code Fix |
|--------|------------|-----------------|----------|
| DL5001 | Performance | Warning         | Yes      |

## Why

`IEnumerable<T>` represents a lazy sequence. Enumerating it multiple times can:

- **Re-execute queries** — database or HTTP calls run again
- **Repeat computation** — generator methods evaluate from scratch
- **Cause side effects** — stateful enumerators may produce different results
- **Waste resources** — the same work is done twice for no benefit

Materialising with `.ToList()` or `.ToArray()` evaluates the sequence once and caches the result.

## What counts as enumeration

- `foreach` loops
- LINQ method calls (`Any()`, `Count()`, `First()`, `Where()`, `Select()`, `ToList()`, `ToArray()`, etc.)
- LINQ query syntax (`from x in items select x`)
- `GetEnumerator()` calls

A single chained LINQ expression like `items.Where(...).Select(...).ToList()` counts as **one** enumeration.

## Examples

### Violation

```csharp
public void Process(IEnumerable<int> items)
{
    var count = items.Count(); // DL5001
    foreach (var x in items) { } // DL5001
}
```

```csharp
public void Process(IEnumerable<int> items)
{
    if (items.Any()) // DL5001
    {
        var first = items.First(); // DL5001
    }
}
```

### Fixed — materialise first

```csharp
public void Process(IEnumerable<int> items)
{
    var itemsList = items.ToList();
    var count = itemsList.Count;
    foreach (var x in itemsList) { }
}
```

### Local variables

The rule also applies to local variables declared as `IEnumerable<T>`:

```csharp
public void Process()
{
    IEnumerable<int> items = GetItems();
    var count = items.Count(); // DL5001
    var list = items.ToList(); // DL5001
}
```

### Constructors

```csharp
public class Processor
{
    public Processor(IEnumerable<int> items)
    {
        var count = items.Count(); // DL5001
        var list = items.ToList(); // DL5001
    }
}
```

### No diagnostic

```csharp
// Single enumeration
public void Process(IEnumerable<int> items)
{
    foreach (var x in items) { }
}

// Concrete collection type — safe to enumerate multiple times
public void Process(List<int> items)
{
    var count = items.Count;
    foreach (var x in items) { }
}

// Chained LINQ counts as single enumeration
public void Process(IEnumerable<int> items)
{
    var result = items.Where(x => x > 5).Select(x => x * 2).ToList();
}

// Nested scope — lambda enumeration is independent
public void Process(IEnumerable<int> items)
{
    var list = items.ToList();
    Action a = () => { foreach (var x in items) { } };
}
```

### Concrete types that are safe

`List<T>`, `T[]`, `IList<T>`, `ICollection<T>`, and other concrete collection types do not trigger the diagnostic.

## Suppression

Apply the `[SuppressMultipleEnumeration]` attribute to parameters that are intentionally enumerated multiple times:

```csharp
using DemoriLabs.Diagnostics.Attributes;

public void Process([SuppressMultipleEnumeration] IEnumerable<int> items)
{
    var count = items.Count(); // No diagnostic
    var list = items.ToList(); // No diagnostic
}
```

## Code fix

**Materialize '{name}' with ToList()** — inserts a `var {name}List = {name}.ToList();` statement at the start of the method body and replaces all references with the materialised variable. Adds `using System.Linq;` if not already present.

## Configuration

```ini
# Disable entirely

```
