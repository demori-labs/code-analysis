# DL3015 - Use null-conditional assignment

Detects if-statements that perform a not-null check and then access a member on the checked variable, which can be simplified using the `?.` null-conditional operator (C# 14).

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3015 | Style    | Warning          | Yes      |

## Why

C# 14 introduces null-conditional assignment, allowing `x?.Prop = value;` instead of guarding with an if-statement. Replacing verbose not-null-check-then-member-access patterns with `?.` reduces noise and makes the intent immediately clear.

## When the diagnostic fires

- `if (x is not null) x.Prop = value;` -- pattern matching not-null check with property assignment
- `if (x != null) x.Prop = value;` -- inequality not-null check with property assignment
- `if (x is not null) x.Method(args);` -- method call on checked variable
- `if (x != null) { x.Prop = value; }` -- block body with single statement
- `if (x is not null) x.Count += 1;` -- compound assignment
- `if (x is not null) x.Inner.Prop = value;` -- nested member access

## When the diagnostic does NOT fire

- The if-statement has an else clause
- The if-body contains multiple statements
- The condition is a null check (`x == null` or `x is null`) rather than a not-null check
- The variable in the condition does not match the receiver of the member access
- The body uses increment (`++`) or decrement (`--`) operators (not supported by C# 14)
- The body is a plain variable assignment without member access (`x = value`)
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
if (x is not null) x.Prop = 42;        // DL3015
if (x != null) x.Prop = 42;            // DL3015
if (x is not null) x.DoWork(5);        // DL3015
if (x != null) { x.Prop = 42; }        // DL3015
if (x is not null) x.Count += 1;       // DL3015
```

### Fixed

```csharp
x?.Prop = 42;
x?.DoWork(5);
x?.Count += 1;
```

### No diagnostic

```csharp
// Has else clause
if (x != null) x.Prop = 42; else DoStuff();

// Multiple statements in body
if (x != null) { x.Prop = 1; x.Other = 2; }

// Null check (reversed logic)
if (x == null) x.Prop = 42;

// Different variables
if (x != null) y.Prop = 42;

// Increment (not supported by C# 14)
if (x is not null) x.Count++;

// Plain variable assignment (DL3014 domain)
if (x != null) x = new C();
```

## Code fix

Replaces the entire if-statement with the body statement, substituting the first `variable.` with `variable?.`. Preserves leading trivia from the original if-statement.
