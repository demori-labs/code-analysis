# DL2003 - Primary constructor parameter should be [ReadOnly]

Suggests adding the `[ReadOnly]` attribute to primary constructor parameters on classes to prevent accidental reassignment.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL2003 | Usage    | Warning          | Yes      |

## Why

Class primary constructor parameters can be reassigned anywhere in the class body, which is often unintentional. Marking them `[ReadOnly]` enables DL2001 enforcement, catching reassignment at compile time.

## Scope

This rule **only** applies to class primary constructor parameters. It does not apply to:

- Record primary constructors (already immutable by design)
- Record struct primary constructors (separate concern)
- Struct primary constructors
- Regular method parameters
- Parameters with `ref`, `out`, or `in` modifiers
- Parameters already marked `[ReadOnly]`

## Examples

### Diagnostic

```csharp
// DL2003: Consider adding [ReadOnly] to primary constructor parameter 'count'
public class Widget(int count);

// DL2003 on each parameter
public class Config(string name, int retries, bool verbose);
```

### No diagnostic

```csharp
using DemoriLabs.CodeAnalysis.Attributes;

// Already has [ReadOnly]
public class Widget([ReadOnly] int count);

// Record — not applicable
public record Person(string Name);

// Regular method parameter — not applicable
public class C
{
    public void M(int x) { }
}
```

## Configuration

```ini
# Adjust severity
dotnet_diagnostic.DL2003.severity = suggestion

# Disable entirely
dotnet_diagnostic.DL2003.severity = none
```

## Code fix

**Add [ReadOnly] attribute** — adds `[ReadOnly]` to the parameter and inserts `using DemoriLabs.CodeAnalysis.Attributes;` if not already present.

```csharp
// Before
public class Widget(int count);

// After
using DemoriLabs.CodeAnalysis.Attributes;

public class Widget([ReadOnly] int count);
```
