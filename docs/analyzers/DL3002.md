# DL3002 - Invert 'if' statement to reduce nesting

Suggests inverting an `if` statement that wraps the remaining body of a method into an early return guard, reducing nesting depth.

| Rule   | Category | Default Severity | Code Fix |
|--------|----------|-----------------|----------|
| DL3002 | Style    | Info            | Yes      |

## Why

Deeply nested code is harder to read and maintain. When an `if` statement is the last statement in a method and wraps the entire remaining body, it can be inverted into a guard clause with an early return:

```csharp
// Before: nested logic
public void Process(Order? order)
{
    if (order is not null)
    {
        Validate(order);
        Submit(order);
        NotifyCustomer(order);
    }
}

// After: flat, linear flow
public void Process(Order? order)
{
    if (order is null)
        return;

    Validate(order);
    Submit(order);
    NotifyCustomer(order);
}
```

Guard clauses make the "exit early" intent explicit and keep the main logic at a lower nesting level.

## When the diagnostic fires

DL3002 is reported in either of these scenarios:

### Pattern 1: Last statement in void context

1. The `if` statement has **no `else`** clause
2. The `if` body is a **block** (`{ ... }`) with at least one statement
3. The `if` is the **last statement** in its parent block
4. The parent block belongs to a **void-returning context**:
   - `void` method
   - `async Task` or `async ValueTask` method
   - Constructor or destructor
   - Property `set`/`init` accessor
   - Event `add`/`remove` accessor
   - `void` local function

### Pattern 2: Followed by a return statement

1. The `if` statement has **no `else`** clause
2. The `if` body is a **block** with **at least 2 statements**
3. The `if` is followed **immediately by a `return` statement** (second-to-last in the block)
4. Works with **any return type** — the code fix uses the following return's value as the early return

## When the diagnostic does NOT fire

- The `if` has an `else` clause
- The `if` is not the last or second-to-last statement
- The containing method returns a value and the if is the last statement (no return to use)
- The `if` body is a single statement without braces
- The `if` body is an empty block
- The `if` is followed by a return but the body only has one statement (minimal nesting benefit)

## Examples

### Violation

```csharp
public void Save(string? path)
{
    if (path is not null) // DL3002
    {
        var data = Serialize();
        File.WriteAllText(path, data);
    }
}
```

### Fixed

```csharp
public void Save(string? path)
{
    if (path is null)
        return;

    var data = Serialize();
    File.WriteAllText(path, data);
}
```

### Violation (followed by return)

```csharp
public int Calculate(bool condition)
{
    if (condition) // DL3002
    {
        DoSomething();
        return 42;
    }

    return 0;
}
```

### Fixed (followed by return)

```csharp
public int Calculate(bool condition)
{
    if (!condition)
        return 0;

    DoSomething();
    return 42;
}
```

### No diagnostic

```csharp
// Has else clause
public void Process(bool condition)
{
    if (condition)
    {
        DoA();
    }
    else
    {
        DoB();
    }
}

// Not the last statement
public void Process(bool condition)
{
    if (condition)
    {
        DoA();
    }

    DoB();
}

// Single statement followed by return — minimal benefit
public int Calculate(bool condition)
{
    if (condition)
    {
        return 42;
    }

    return 0;
}
```

## Code fix

**Invert 'if' to reduce nesting** — negates the condition, adds an early `return;`, and moves the original body statements out of the block.

The code fix intelligently negates conditions:

| Original condition | Negated condition |
|-------------------|-------------------|
| `condition`       | `!condition`      |
| `!condition`      | `condition`       |
| `x == 0`          | `x != 0`          |
| `x != null`       | `x == null`       |
| `x > 0`           | `x <= 0`          |
| `x is null`       | `x is not null`   |
| `x is not null`   | `x is null`       |
| `true`            | `false`           |
| `a && b`          | `!(a && b)`       |
