# DL3002 - Invert 'if' statement to reduce nesting

Suggests inverting an `if` statement that wraps the remaining body of a method into an early return guard, reducing nesting depth.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3002 | Style    | Info             | Yes      |

## Why

Deeply nested code is harder to read and maintain. When an `if` statement is the last statement in a method and wraps the entire remaining body, it can be inverted into a guard clause with an early return:

```csharp
// Before: nested logic
public void Process(Order? order)
{
    if (order is not null)
    {
        Validate(order);
        Submit(order);
        NotifyCustomer(order);
    }
}

// After: flat, linear flow
public void Process(Order? order)
{
    if (order is null)
        return;

    Validate(order);
    Submit(order);
    NotifyCustomer(order);
}
```

Guard clauses make the "exit early" intent explicit and keep the main logic at a lower nesting level.

## When the diagnostic fires

DL3002 is reported when an `if` statement meets these conditions:

1. The `if` body is a **block** (`{ ... }`) with at least one statement
2. The `if` is **not** an inner `if` of an `else if` chain (only the outermost `if` is flagged)
3. The `if` is inside a **block** or **switch section**
4. One of the following structural patterns applies:

### Pattern 1: Last statement in void or continue context

- The `if` has **no `else`** clause
- The `if` is the **last statement** in its parent block
- The parent block belongs to a **void-returning** or **loop** context:
    - `void` method, `async Task` or `async ValueTask` method
    - Constructor or destructor
    - Property/event accessor
    - Lambda or anonymous method
    - `void` local function
    - `foreach`, `for`, `while`, or `do` loop body (exit via `continue`)

### Pattern 2: Second-to-last, followed by an exit statement

- The `if` has **no `else`** clause
- The `if` is the **second-to-last** statement in its parent block
- The next statement is an exit statement (`return`, `throw`, `break`, `continue`, or `yield break`)
- Works in **any return context** — the code fix uses the following exit statement

### Pattern 3: Else chain ending with exits

- The `if` **has** an `else` clause
- The entire `else`/`else if` chain ends with exit statements (`return`, `throw`, `break`, `continue`, or `yield break`)

## When the diagnostic does NOT fire

- The `if` is part of an inner `else if` — only the outermost `if` is flagged
- The `if` is not the last or second-to-last statement (and has no qualifying `else` chain)
- The containing method returns a value and the `if` is the last statement with no `else` (no return to use)
- The `if` body is a single statement without braces
- The `if` body is an empty block

## Supported contexts

| Context                                 | Exit strategy                |
| --------------------------------------- | ---------------------------- |
| `void` method                           | `return;`                    |
| `async Task` / `async ValueTask` method | `return;`                    |
| Constructor / destructor                | `return;`                    |
| Property `set`/`init` accessor          | `return;`                    |
| Lambda / anonymous method               | `return;`                    |
| `void` local function                   | `return;`                    |
| `foreach` / `for` / `while` / `do` loop | `continue;`                  |
| Value-returning method                  | `return <value>;`            |
| `switch` section                        | `return <value>;` / `break;` |

## Examples

### Violation — void method

```csharp
public void Save(string? path)
{
    if (path is not null) // DL3002
    {
        var data = Serialize();
        File.WriteAllText(path, data);
    }
}
```

### Fixed — void method

```csharp
public void Save(string? path)
{
    if (path is null)
        return;

    var data = Serialize();
    File.WriteAllText(path, data);
}
```

### Violation — followed by return

```csharp
public int Calculate(bool condition)
{
    if (condition) // DL3002
    {
        DoSomething();
        return 42;
    }

    return 0;
}
```

### Fixed — followed by return

```csharp
public int Calculate(bool condition)
{
    if (!condition)
        return 0;

    DoSomething();
    return 42;
}
```

### Violation — loop body

```csharp
public void NotifyAll(IEnumerable<User> users)
{
    foreach (var user in users)
    {
        if (user.IsActive) // DL3002
        {
            SendEmail(user);
            Log(user.Name);
        }
    }
}
```

### Fixed — loop body

```csharp
public void NotifyAll(IEnumerable<User> users)
{
    foreach (var user in users)
    {
        if (!user.IsActive)
            continue;

        SendEmail(user);
        Log(user.Name);
    }
}
```

### Violation — else chain ending with exits

```csharp
public void Process(string? input)
{
    if (input is not null) // DL3002
    {
        DoWork(input);
        Save(input);
    }
    else
    {
        throw new ArgumentNullException(nameof(input));
    }
}
```

### No diagnostic

```csharp
// Not the last statement and no exit follows
public void Process(bool condition)
{
    if (condition)
    {
        DoA();
    }

    DoB();
}

// Else chain does NOT end with an exit
public void Process(bool condition)
{
    if (condition)
    {
        DoA();
    }
    else
    {
        DoB(); // no return/throw — cannot invert safely
    }
}

// Value-returning method with if as last statement and no else — no return to use
public int Calculate(bool condition)
{
    DoSetup();

    if (condition)
    {
        return 42;
    }
}
```

## Code fix

**Invert 'if' to reduce nesting** — negates the condition, adds an early exit statement, and moves the body statements out of the block. Recursively flattens nested invertible `if` statements.

### Condition negation

Uses pattern matching (`is`/`is not`) over traditional operators where possible.

| Original                      | Negated                           |
| ----------------------------- | --------------------------------- |
| `x != null`                   | `x is null`                       |
| `x == null`                   | `x is not null`                   |
| `x is not null`               | `x is null`                       |
| `x is null`                   | `x is not null`                   |
| `x is string`                 | `x is not string`                 |
| `x is string { Length: > 0 }` | `x is not string { Length: > 0 }` |
| `condition`                   | `condition is false`              |
| `x.IsEnabled`                 | `x.IsEnabled is false`            |
| `IsReady()`                   | `IsReady() is false`              |
| `x == 0`                      | `x is not 0`                      |
| `x != 0`                      | `x is 0`                          |
| `x.HasValue`                  | `x is null`                       |
| `!condition`                  | `condition`                       |
| `true`                        | `false`                           |
| `false`                       | `true`                            |
| `x > 0`                       | `x <= 0`                          |
| `x >= 0`                      | `x < 0`                           |
| `x < 0`                       | `x >= 0`                          |
| `x <= 0`                      | `x > 0`                           |
| `a == b` (two non-constants)  | `a != b`                          |
| `await IsReadyAsync()`        | `!await IsReadyAsync()`           |

`&&` and `||` are negated via De Morgan's law, applied recursively:

| Original                                     | Negated                                                  |
| -------------------------------------------- | -------------------------------------------------------- |
| `a && b`                                     | `a is false \|\| b is false`                             |
| `a \|\| b`                                   | `a is false && b is false`                               |
| `x != null && x.IsEnabled && !x.IsCompleted` | `x is null \|\| x.IsEnabled is false \|\| x.IsCompleted` |
| `(a && b) \|\| c`                            | `(a is false \|\| b is false) && c is false`             |

### Equality rewriting

`==` and `!=` comparisons against literals and compile-time constants are rewritten to pattern matching in both the guard and the extracted body:

| Original                        | Rewritten                           |
| ------------------------------- | ----------------------------------- |
| `x == null`                     | `x is null`                         |
| `x != null`                     | `x is not null`                     |
| `x == 0`                        | `x is 0`                            |
| `x != 0`                        | `x is not 0`                        |
| `x == "hello"`                  | `x is "hello"`                      |
| `state == ConnectionState.Open` | `state is ConnectionState.Open`     |
| `state != ConnectionState.Open` | `state is not ConnectionState.Open` |
