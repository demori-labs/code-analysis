# DL3010 - Simplify conditional return to ternary

Detects `if` statements that return different values in both branches and suggests simplifying to a single ternary return.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3010 | Style    | Info             | Yes      |

## Why

An `if` statement that returns a value in one branch and a different value in the other is equivalent to a ternary conditional expression. Simplifying removes unnecessary control flow and makes the intent clearer.

## When the diagnostic fires

- `if (cond) { return a; } return b;` -- fall-through (no else, next statement is a return)
- `if (cond) return a; else return b;` -- with else clause, no braces
- `if (cond) { return a; } else { return b; }` -- with else clause, block bodies
- Return values are method calls, new expressions, or other short expressions

## When the diagnostic does NOT fire

- Void returns (`if (cond) return; else return;`)
- Both return values are boolean literals (`true`/`false`) -- that's DL3008's domain
- Either return value contains a nested ternary expression
- Either return value text is longer than 60 characters
- Intermediate statements between `if` and the fall-through return
- No matching else or fall-through return
- Multiple statements in the if body
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
if (cond) { return 1; } return 2;             // DL3010
if (cond) return Foo(); else return Bar();     // DL3010
if (a && b) { return 1; } else { return 2; }  // DL3010
```

### Fixed

```csharp
return cond ? 1 : 2;
return cond ? Foo() : Bar();
return a && b ? 1 : 2;
```

### No diagnostic

```csharp
// Void return
if (cond) return; else return;

// Boolean literals (DL3008)
if (cond) return true; else return false;

// Nested ternary
if (cond) return x ? 1 : 2; else return 3;

// Long return value
if (cond) return SomeVeryLongMethodName(withManyParameters, andMore); else return 0;

// Intermediate statement
if (cond) { return 1; }
DoStuff();
return 2;
```

## Code fix

Replaces the entire `if` construct (including the fall-through return statement when applicable) with `return condition ? trueValue : falseValue;`. Preserves leading trivia from the original `if` statement.
