# DL3019 - Use file-scoped namespace declaration

Flags block-scoped namespace declarations that can be converted to file-scoped namespace declarations.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3019 | Style    | Warning          | Yes      |

## Why

File-scoped namespaces (introduced in C# 10) eliminate one level of indentation across the entire file, reducing visual noise and making the code easier to scan. Since the vast majority of files contain a single namespace, the block-scoped form adds boilerplate `{` and `}` braces with no benefit.

## When the diagnostic fires

- A file contains one or more block-scoped `namespace` declarations that all resolve to the same namespace
- A file contains nested namespaces where the outer namespace is a pure wrapper (no direct members):
  - `namespace A { namespace B { class C {} } }` is flagged because it can become `namespace A.B;`
  - Deeply nested wrappers like `namespace A { namespace B { namespace C { ... } } }` are also flagged

## When the diagnostic does NOT fire

- The file already uses a file-scoped namespace declaration
- Multiple block namespaces with different names exist (DL3020 handles this)
- A nested namespace where the outer namespace has its own direct members (DL3020 handles this)
- The file has no namespace declaration

## Examples

### Violation

```csharp
namespace MyApp // DL3019
{
    public class UserService
    {
        public void CreateUser(string name) { }
    }
}
```

```csharp
namespace MyApp // DL3019
{
    namespace Models
    {
        public class User
        {
            public int Id { get; set; }
        }
    }
}
```

### Fixed

```csharp
namespace MyApp;

public class UserService
{
    public void CreateUser(string name) { }
}
```

```csharp
namespace MyApp.Models;

public class User
{
    public int Id { get; set; }
}
```

### No diagnostic

```csharp
// Already file-scoped
namespace MyApp;

public class Program { }
```

## Overlap with IDE0161

Microsoft ships [IDE0161](https://learn.microsoft.com/en-us/dotnet/fundamentals/code-analysis/style-rules/ide0161), a built-in code-style rule that performs a similar check. DL3019 suppresses IDE0161 automatically via a `DiagnosticSuppressor` to avoid duplicate diagnostics. See [DL3018's overlap section](DL3018.md#overlap-with-ide0130) for the rationale — the same limitations apply (IDE-only by default, requires `EnforceCodeStyleInBuild` for CI).

## Code fix

Converts block-scoped namespace declarations to a single file-scoped declaration. For nested namespaces, the names are concatenated with `.` (e.g. `namespace A { namespace B { } }` becomes `namespace A.B;`). All members are dedented to match the reduced nesting level. Multiple blocks with the same name are merged into one.
