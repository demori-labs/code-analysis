# DL2001 / DL2002 - ReadOnly parameter enforcement

These two rules work together to enforce the `[ReadOnly]` attribute on parameters.

| Rule   | Category | Default Severity | Description                                              |
| ------ | -------- | ---------------- | -------------------------------------------------------- |
| DL2001 | Usage    | Error            | Parameter marked `[ReadOnly]` must not be reassigned     |
| DL2002 | Usage    | Error            | `[ReadOnly]` is incompatible with the parameter modifier |

## DL2001 — Parameter must not be reassigned

Reports when a `[ReadOnly]` parameter is reassigned within the method body. This includes simple assignments, compound assignments, and increment/decrement operations.

### Violation

```csharp
using DemoriLabs.Diagnostics.Attributes;

public void Process([ReadOnly] int count)
{
    count = 10; // DL2001
    count += 5; // DL2001
    count++; // DL2001
}
```

### No diagnostic

```csharp
public void Process([ReadOnly] int count)
{
    var local = count + 1; // OK — reading the parameter
    Console.WriteLine(count); // OK — passing as argument
}
```

### Primary constructor parameters

The rule also applies to primary constructor parameters on classes and record structs:

```csharp
public class Widget([ReadOnly] int count)
{
    public void Reset()
    {
        count = 0; // DL2001
    }
}
```

### Code fix

**Introduce local variable for '{name}'** — creates a new local variable and redirects all subsequent references:

```csharp
// Before
public void Process([ReadOnly] int count)
{
    count += 5;
    Console.WriteLine(count);
}

// After
public void Process([ReadOnly] int count)
{
    var countLocal = count + 5;
    Console.WriteLine(countLocal);
}
```

Compound assignments are expanded: `x += 5` becomes `var xLocal = x + 5`. Increment/decrement operations are expanded: `x++` becomes `var xLocal = x + 1`.

## DL2002 — Incompatible modifier

Reports when `[ReadOnly]` is applied to a parameter where it conflicts with the parameter's semantics.

### Incompatible cases

| Parameter kind                             | Why incompatible                      |
| ------------------------------------------ | ------------------------------------- |
| `ref`                                      | Exists specifically to allow mutation |
| `out`                                      | Must be assigned by the callee        |
| `in`                                       | Already read-only by the language     |
| Record class primary constructor           | Parameters are immutable by design    |
| Readonly record struct primary constructor | Parameters are immutable by design    |

### Violation

```csharp
using DemoriLabs.Diagnostics.Attributes;

public void M([ReadOnly] ref int x) { } // DL2002
public void N([ReadOnly] out int x) { x = 0; } // DL2002
public void P([ReadOnly] in int x) { } // DL2002

// Record class — already immutable
public record Person([ReadOnly] string Name); // DL2002

// Readonly record struct — already immutable
public readonly record struct Point([ReadOnly] int X); // DL2002
```

### Allowed on mutable record struct

```csharp
// Record struct is mutable — [ReadOnly] is valid here
public record struct Counter([ReadOnly] int Value); // No DL2002
```

### Code fix

**Remove `[ReadOnly]` attribute** — removes the conflicting attribute from the parameter.

## Configuration

```ini
# Adjust severity
dotnet_diagnostic.DL2001.severity = warning
dotnet_diagnostic.DL2002.severity = warning

# Disable entirely
dotnet_diagnostic.DL2001.severity = none
dotnet_diagnostic.DL2002.severity = none
```
