# DL3003 - Use constant pattern

Detects equality comparisons against compile-time constants (`== null`, `== 42`, `== MyEnum.Value`) that should use pattern matching (`is null`, `is 42`, etc.).

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3003 | Style    | Warning          | Yes      |

## Why

Pattern matching with `is` is safer and more idiomatic in modern C#. For null checks, `is null` bypasses any custom `operator ==` overloads, performing a direct reference equality check. For all constants, `is` patterns are more readable and consistent with modern C# style.

## When the diagnostic fires

- `x == null` or `null == x`
- `x != null` or `null != x`
- `x == true`, `x == false`, `x != true`, `x != false`
- `x == 42`, `x != 0`, `42 == x` (any numeric literal)
- `x == 'a'` (char literals)
- `x == MyEnum.Value` (enum members)
- `x == SomeConstant` (const fields and locals)
- Negative literals: `x == -1`

## When the diagnostic does NOT fire

- Neither operand is a compile-time constant (e.g. `a == b`)
- Both operands are constants (e.g. `1 == 1`)
- String literal comparisons (e.g. `s == "hello"`) — handled by a dedicated string comparison analyser
- Inside `Expression<Func<>>` lambdas (pattern matching is not supported in expression trees)
- The comparison is part of a `||`/`&&` chain that [DL3005](DL3005.md) handles (e.g. `id == null || id == 0`) — DL3005 takes priority
- The comparison is wrapped in an outer comparison or negation (e.g. `(o == null) == false`, `!(o == null)`) — the outer expression takes priority

## Examples

### Violation

```csharp
if (obj == null) { }           // DL3003
if (obj != null) { }           // DL3003
if (x == 42) { }               // DL3003
if (status == Status.Active) { } // DL3003
if (null == obj) { }           // DL3003
```

### Fixed

```csharp
if (obj is null) { }
if (obj is not null) { }
if (x is 42) { }
if (status is Status.Active) { }
if (obj is null) { }
```

### No diagnostic

```csharp
// Already using pattern matching
if (obj is null) { }

// Two variables — not a constant comparison
if (a == b) { }

// Inside expression tree
dbContext.Users.Where(u => u.Name == null);
```

## Code fix

Replaces the equality operator with a pattern matching expression. `==` becomes `is`, `!=` becomes `is not`. Reversed operands (`null == x`) are flipped to `x is null`. Negative numeric literals are parenthesised: `x is (-1)`.
