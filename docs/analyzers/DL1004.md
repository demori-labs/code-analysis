# DL1004 - Data class could be a record

Classes that only define data members (properties and constructors) with no behaviour are better expressed as records with immutable properties.

| Rule   | Category | Default Severity | Code Fix |
|--------|----------|-----------------|----------|
| DL1004 | Design   | Info            | Yes      |

## Why

Records provide value-based equality, concise syntax, and immutability by convention. A class that only holds data gains all of these benefits for free by becoming a record.

## Detection criteria

The diagnostic is reported when **all** of the following are true:

- The type is a `class` (not a struct, interface, or record)
- It is not `static`, `abstract`, or `partial`
- It inherits directly from `System.Object` (no custom base class)
- It contains at least one property
- It contains no methods, fields, events, operators, conversions, or destructors
- It does not have the `[Mutable]` attribute

## Examples

### Violation

```csharp
// DL1004: Class 'Person' only defines data members
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}
```

### Fixed

```csharp
public record Person
{
    public required string Name { get; init; }
    public required int Age { get; init; }
}
```

### Class with constructor

```csharp
// DL1004: still only data — constructors are allowed
public class Person
{
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }

    public string Name { get; set; }
    public int Age { get; set; }
}
```

### Generic class

```csharp
// DL1004
public class Wrapper<T>
{
    public T Value { get; set; }
}
```

### Class implementing interface

```csharp
// DL1004: interfaces are fine, only base classes are excluded
public class Person : IEntity
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```

### No diagnostic

```csharp
// Has a method — not a pure data class
public class Person
{
    public string Name { get; set; }
    public string GetDisplayName() => Name.ToUpper();
}

// Has a field
public class Person
{
    private readonly int _id;
    public string Name { get; set; }
}

// Abstract class
public abstract class Entity
{
    public int Id { get; set; }
}

// Has a base class
public class Person : BaseEntity
{
    public string Name { get; set; }
}

// Partial class
public partial class Person
{
    public string Name { get; set; }
}

// Already a record
public record Person
{
    public string Name { get; init; }
}
```

## Suppression

The `[Mutable]` attribute can be used to suppress this diagnostic for classes that are intentionally mutable and should remain as classes. Use this sparingly — if the class truly needs mutability, consider whether a mutable record struct would be more appropriate.

```csharp
using DemoriLabs.Diagnostics.Attributes;

[Mutable]
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}
```

## Configuration

```ini
# Adjust severity
dotnet_diagnostic.DL1004.severity = warning

# Disable entirely
dotnet_diagnostic.DL1004.severity = none
```

## Code fix

**Convert to record** — transforms the class into a record with immutable properties:

- Replaces `class` with `record`
- Converts `set` accessors to `init`
- Adds `required` to properties without default values
- Preserves constructors, type parameters, interfaces, attributes, and all other members
