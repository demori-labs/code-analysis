# DL3009 - Simplify boolean assignment

Detects `if` statements that assign `true`/`false` in both branches and suggests simplifying to a single assignment statement.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3009 | Style    | Warning          | Yes      |

## Why

An `if` statement that assigns `true` in one branch and `false` in the other is equivalent to assigning the condition itself (or its negation). Simplifying removes unnecessary control flow and makes the intent clearer.

## When the diagnostic fires

- `if (cond) x = true; else x = false;` -- inline bodies
- `if (cond) { x = true; } else { x = false; }` -- block bodies
- `if (cond) x = false; else x = true;` -- negated
- Assignment to properties: `if (cond) obj.Flag = true; else obj.Flag = false;`
- Assignment to fields: `if (cond) _flag = true; else _flag = false;`

## When the diagnostic does NOT fire

- No else clause (`if (cond) x = true;` alone)
- Non-boolean assignment values (`if (cond) x = 1; else x = 0;`)
- Different assignment targets (`if (cond) x = true; else y = false;`)
- Both branches assign the same value (`x = true`/`x = true`)
- Multiple statements in the if body
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
if (cond) x = true; else x = false;         // DL3009
if (cond) { x = true; } else { x = false; } // DL3009
if (cond) x = false; else x = true;          // DL3009
```

### Fixed

```csharp
x = cond;
x = cond;
x = cond is false;
```

### No diagnostic

```csharp
// No else clause
if (cond) x = true;

// Non-boolean values
if (cond) x = 1; else x = 0;

// Different targets
if (cond) x = true; else y = false;

// Same value both branches
if (cond) x = true; else x = true;
```

## Code fix

When the `if` assigns `true` and the else assigns `false`, replaces the entire construct with `x = condition;`. When the `if` assigns `false` and the else assigns `true`, replaces with `x = condition is false;`. Preserves leading trivia from the original `if` statement.
