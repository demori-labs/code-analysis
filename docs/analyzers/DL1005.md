# DL1005 - Type can use a primary constructor

Classes and structs with a single constructor that only assigns parameters to readonly fields can be refactored to use a primary constructor.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL1005 | Design   | Info             | Yes      |

## Why

- Primary constructors reduce boilerplate by eliminating explicit field declarations and constructor bodies
- The code fix adds `[ReadOnly]` to each primary constructor parameter, ensuring immutability is enforced by the existing `ReadOnlyParameter` analyzer (DL2001)
- This rule replaces the built-in IDE0290 suggestion via a `DiagnosticSuppressor`, because IDE0290 does not add `[ReadOnly]` attributes

## Examples

### Violation

```csharp
public class MyService
{
    private readonly ILogger _logger;
    private readonly IRepository _repository;

    public MyService(ILogger logger, IRepository repository) // DL1005
    {
        _logger = logger;
        _repository = repository;
    }

    public void DoWork() => _logger.Log("Working");
}
```

### Fixed

```csharp
using DemoriLabs.CodeAnalysis.Attributes;

public class MyService([ReadOnly] ILogger logger, [ReadOnly] IRepository repository)
{
    public void DoWork() => logger.Log("Working");
}
```

### No diagnostic

```csharp
// Constructor with logic beyond simple field assignment
public class MyService
{
    private readonly ILogger _logger;

    public MyService(ILogger logger)
    {
        _logger = logger ?? throw new ArgumentNullException(nameof(logger));
    }
}

// Multiple constructors
public class MyService
{
    private readonly int _id;

    public MyService(int id) { _id = id; }
    public MyService() : this(0) { }
}

// Already uses a primary constructor
public class MyService(int id)
{
    public int Id => id;
}
```

## Code fix

The code fix converts the constructor to a primary constructor:

1. Moves constructor parameters to the type declaration with `[ReadOnly]` attributes
2. Removes the constructor body and backing field declarations
3. Replaces all field references throughout the type with the corresponding parameter names (stripping `_` and `m_` prefixes)
4. Preserves `: base(...)` initialiser calls as primary constructor base type syntax
5. Adds `using DemoriLabs.CodeAnalysis.Attributes;` if not already present
