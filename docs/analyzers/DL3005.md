# DL3005 - Use logical pattern

Detects chained comparisons, pattern checks, and `Nullable<T>.HasValue` / `.Value` accessors on the same variable that can be replaced with a single logical pattern (`is X or Y`, `is >= A and < B`, `is not X and not Y`).

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3005 | Style    | Warning          | Yes      |

## Priority over DL3003 and DL3004

DL3005 has the highest priority among the pattern matching analyzers. When DL3005 handles a `||`/`&&` chain, the individual leaves are **suppressed** — DL3003 (constant pattern) and DL3004 (negation pattern) check whether their node is part of a chain DL3005 would handle, and skip if so. This prevents overlapping diagnostics.

```
DL3005 (logical chains)  ← DL3003 and DL3004 defer to this
DL3003 (single comparison, or outer wrapped expression) ← inner DL3003 defers to outer DL3003/DL3004
DL3004 (negation)        ← independent
```

## Why

Logical patterns express constraints on a single variable concisely and readably. `x is 1 or 2 or 3` is clearer than `x == 1 || x == 2 || x == 3` — the variable appears once instead of being repeated in every comparison. Range checks like `x is >= 0 and < 100` read naturally as a bounded interval.

## When the diagnostic fires

### `||` chains (or patterns)

- `x == 1 || x == 2 || x == 3` — equality chain
- `id == null || id == 0` — nullable null-or-value
- `id == null || id > 10` — mixed equality and relational
- `!id.HasValue || id == 0` — `Nullable<T>.HasValue` negation treated as `== null`
- `id is null || id is 0` — already pattern matching but separate `is` expressions

### `&&` chains (and patterns)

- `x != 1 && x != 2` — not-equals chain
- `x >= 0 && x < 100` — relational range
- `id != null && id > 0` — mixed not-equals and relational
- `id.HasValue && id != 0` — `Nullable<T>.HasValue` treated as `!= null`
- `id != null && id.Value > 0` — `Nullable<T>.Value` accessor recognised as same variable
- `id is not null && id is not 0` — separate `is` expressions merged
- `id is not null && id is > 0` — mixed `is` patterns merged
- `x >= 0 && x < 100 && x != 50` — three or more leaves

All comparisons must reference the same symbol and compare against compile-time constants.

### `Nullable<T>` support

- `HasValue` and `!HasValue` are recognised as null/not-null checks on the nullable variable
- `.Value` accessor is resolved to the underlying nullable variable (e.g. `id.Value > 0` matches `id != null`)
- Semantic check ensures only `System.Nullable<T>` triggers this — custom types with `HasValue` or `Value` properties are not affected

## When the diagnostic does NOT fire

- Different variables on each side (`x == 1 || y == 2`)
- Non-constant right-hand sides (`x == a || x == b`)
- String literal comparisons (`s == "a" || s == "b"`)
- Inside `Expression<Func<>>` lambdas
- Already using combined pattern matching (`x is 1 or 2 or 3`)
- Custom types with `HasValue` property (not `Nullable<T>`)
- Custom types with `Value` property (not `Nullable<T>`)

## Examples

### Violation

```csharp
// || chains
if (x == 1 || x == 2 || x == 3) { }    // DL3005
if (id == null || id == 0) { }          // DL3005
if (id == null || id > 10) { }          // DL3005
if (!id.HasValue || id == 0) { }        // DL3005
if (id is null || id is 0) { }          // DL3005

// && chains
if (x >= 0 && x < 100) { }             // DL3005
if (x != 1 && x != 2) { }              // DL3005
if (id != null && id > 0) { }          // DL3005
if (id.HasValue && id != 0) { }        // DL3005
if (id != null && id.Value > 0) { }    // DL3005
if (id is not null && id is not 0) { } // DL3005
if (id is not null && id is > 0) { }   // DL3005
```

### Fixed

```csharp
// || chains
if (x is 1 or 2 or 3) { }
if (id is null or 0) { }
if (id is null or > 10) { }
if (id is null or 0) { }
if (id is null or 0) { }

// && chains
if (x is >= 0 and < 100) { }
if (x is not 1 and not 2) { }
if (id is not null and > 0) { }
if (id is not null and not 0) { }
if (id is not null and > 0) { }
if (id is not null and not 0) { }
if (id is not null and > 0) { }
```

### No diagnostic

```csharp
// Already using combined pattern matching
if (x is 1 or 2 or 3) { }

// Different variables
if (x == 1 || y == 2) { }

// Non-constant
if (x == a || x == b) { }

// Custom type with HasValue — not Nullable<T>
public class Option { public bool HasValue { get; set; } }
if (!opt.HasValue || opt == null) { }
```

## Code fix

Rewrites the entire `||`/`&&` chain as a single `is` pattern expression:

- `||` with `==` → `is X or Y or Z`
- `||` with mixed `==`/relational → `is X or > Y`
- `&&` with `!=` → `is not X and not Y`
- `&&` with relational → `is >= A and < B`
- `&&` with mixed `!=`/relational → `is not X and > Y`
- `HasValue`/`.Value` accessors are normalised to the base variable
