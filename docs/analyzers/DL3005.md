# DL3005 - Use logical pattern

Detects chained comparison operators on the same variable that can be replaced with logical patterns (`is X or Y`, `is >= A and < B`, `is not X and not Y`).

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3005 | Style    | Warning          | Yes      |

## Why

Logical patterns express constraints on a single variable concisely and readably. `x is 1 or 2 or 3` is clearer than `x == 1 || x == 2 || x == 3` — the variable appears once instead of being repeated in every comparison. Range checks like `x is >= 0 and < 100` read naturally as a bounded interval.

## When the diagnostic fires

- `x == 1 || x == 2 || x == 3` — equality chain with `||` (2+ comparisons)
- `x >= 0 && x < 100` — relational range with `&&` (exactly 2 comparisons)
- `x != 1 && x != 2` — not-equals chain with `&&` (2+ comparisons)

All comparisons must reference the same symbol and compare against compile-time constants.

## When the diagnostic does NOT fire

- Different variables on each side (`x == 1 || y == 2`)
- Non-constant right-hand sides (`x == a || x == b`)
- Mixed operators that don't form a coherent pattern (`x == 1 || x > 10`)
- Three or more relational comparisons (`x >= 0 && x < 100 && x != 50`)
- Nested `||`/`&&` inside the other kind
- String literal comparisons (e.g. `s == "a" || s == "b"`) — handled by a dedicated string comparison analyser
- Inside `Expression<Func<>>` lambdas
- Already using pattern matching

## Examples

### Violation

```csharp
if (x == 1 || x == 2 || x == 3) { } // DL3005
if (x >= 0 && x < 100) { }          // DL3005
if (x != 1 && x != 2) { }           // DL3005
```

### Fixed

```csharp
if (x is 1 or 2 or 3) { }
if (x is >= 0 and < 100) { }
if (x is not 1 and not 2) { }
```

### No diagnostic

```csharp
// Already using pattern matching
if (x is 1 or 2 or 3) { }

// Different variables
if (x == 1 || y == 2) { }

// Non-constant
if (x == a || x == b) { }
```

## Code fix

Rewrites the entire `||`/`&&` chain as a single `is` pattern expression. For `||` with `==`, produces `is X or Y or Z`. For `&&` with `!=`, produces `is not X and not Y`. For `&&` with relationals, produces `is >= A and < B`.
