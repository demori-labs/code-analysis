# DL3014 - Use null-coalescing assignment

Detects if-statements that null-check a variable and then assign to the same variable, which can be simplified to the `??=` operator.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3014 | Style    | Warning          | Yes      |

## Why

The null-coalescing assignment operator `??=` is a concise idiom for "assign a value only when the variable is null". Replacing verbose null-check-then-assign patterns with `??=` reduces noise and makes the intent immediately clear.

## When the diagnostic fires

- `if (x == null) x = new Foo();` -- equality null check with simple assignment
- `if (x is null) x = new Foo();` -- pattern matching null check
- `if (x == null) { x = new Foo(); }` -- block body with single assignment
- `if (x == null) x = GetDefault();` -- method call as assigned value
- `if (_field == null) _field = new Foo();` -- field assignment
- `if (obj.Prop == null) obj.Prop = new Foo();` -- property assignment

## When the diagnostic does NOT fire

- The if-statement has an else clause
- The if-body contains multiple statements
- The condition is a not-null check (`x != null`)
- The variable in the condition does not match the assignment target
- The assignment is a compound assignment (`x += 1`)
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
if (x == null) x = new Foo();           // DL3014
if (x is null) x = new Foo();           // DL3014
if (x == null) { x = new Foo(); }       // DL3014
if (_field == null) _field = GetDefault(); // DL3014
```

### Fixed

```csharp
x ??= new Foo();
_field ??= GetDefault();
```

### No diagnostic

```csharp
// Has else clause
if (x == null) x = new Foo(); else DoStuff();

// Multiple statements in body
if (x == null) { x = new Foo(); DoStuff(); }

// Not-null check (reversed logic)
if (x != null) x = new Foo();

// Different variables
if (x == null) y = new Foo();

// Compound assignment
if (x == null) x += 1;
```

## Code fix

Replaces the entire if-statement with `variable ??= value;`. Preserves leading trivia from the original if-statement.
