# DL3013 - Use null-coalescing operator

Detects ternary expressions and if-return patterns that perform null checks and can be simplified to the `??` operator.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3013 | Style    | Warning          | Yes      |

## Why

The null-coalescing operator `??` is a concise idiom for "use this value, or fall back to that value when null". Replacing verbose null checks with `??` reduces noise and makes the intent immediately clear.

## When the diagnostic fires

- `x != null ? x : y` -- ternary with `!=` null check
- `x == null ? y : x` -- ternary with `==` null check (reversed branches)
- `x is not null ? x : y` -- ternary with pattern matching
- `x is null ? y : x` -- ternary with pattern matching (reversed)
- `if (x != null) return x; return y;` -- if-statement fall-through
- `if (x is not null) return x; return y;` -- if-statement with pattern
- `if (x != null) return x; else return y;` -- if-statement with else

Only simple identifiers are flagged. Property accesses and method calls are excluded because they could have side effects or be evaluated differently on a second access.

## When the diagnostic does NOT fire

- The checked expression is a method call (`GetFoo() != null ? GetFoo() : y`)
- The checked expression is a property or member access (`obj.Prop != null ? obj.Prop : y`)
- The variable in the condition does not match the "non-null" branch
- The if-statement has intermediate statements before the fall-through return
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
// Ternary forms
return x != null ? x : y;          // DL3013
return x == null ? y : x;          // DL3013
return x is not null ? x : y;      // DL3013
return x is null ? y : x;          // DL3013

// If-statement forms
if (x != null) return x;           // DL3013
return y;

if (x != null) return x;           // DL3013
else return y;
```

### Fixed

```csharp
return x ?? y;
```

### No diagnostic

```csharp
// Method call - could cause double evaluation
return GetFoo() != null ? GetFoo() : y;

// Property access - could have side effects
return obj.Prop != null ? obj.Prop : y;

// Variable doesn't match non-null branch
return x != null ? y : z;
```

## Code fix

- Ternary: replaces the entire conditional expression with `x ?? y`
- If-statement (fall-through): replaces the if-statement and the next return with `return x ?? y;`
- If-statement (with else): replaces the if-statement with `return x ?? y;`

Preserves leading trivia from the original node.
