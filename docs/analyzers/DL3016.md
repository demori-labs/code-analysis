# DL3016 - Redundant type pattern

Flags `is Type` and `is Type variable` expressions where the compile-time type already guarantees the check always succeeds.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3016 | Style    | Warning          | Yes (nullable reference types only) |

## Why

When the expression's compile-time type is the same as (or derives from) the checked type, the type pattern is redundant:

- For **value types** and **non-nullable reference types**, the check is always true.
- For **nullable reference types** without a variable, the check is just a null check and should use `is not null` instead.

## When the diagnostic fires

- `expr is Type` where the expression's type is already `Type` or a subtype
- `expr is Type variable` where the expression is non-nullable (the variable is just an alias)
- Works for identity conversions (same type) and implicit reference conversions (subtype to base, concrete to interface)

## When the diagnostic does NOT fire

- The type check narrows the type (e.g. `object` to `string`, base to derived)
- `expr is Type variable` where the expression is nullable — this is the correct idiom for null-checking and extracting a non-null typed variable (`is not null s` does not compile)
- Inside `Expression<Func<>>` lambdas
- When nullable reference types are not active for the code (`#nullable disable` or NRT not enabled) — reference type checks are skipped because `is Type` may be an intentional null guard

## Examples

### Violation

```csharp
#nullable enable

void M(int c)
{
    if (c is int) { }          // DL3016 — always true
    if (c is int x) { }        // DL3016 — always true, x is an alias for c
}

void M(string str)
{
    if (str is string) { }     // DL3016 — always true (non-nullable)
    if (str is string s) { }   // DL3016 — always true, s is an alias for str
}

void M(string? str)
{
    if (str is string) { }     // DL3016 — redundant type check, use is not null
}

void M(Dog dog)
{
    if (dog is Animal) { }     // DL3016 — Dog is always Animal
}

void M(List<int> list)
{
    if (list is IEnumerable<int>) { }  // DL3016 — List<int> always implements IEnumerable<int>
}
```

### Fixed

```csharp
#nullable enable

void M(string? str)
{
    if (str is not null) { }   // code fix applied
}
```

### No diagnostic

```csharp
#nullable enable

// Narrowing type check — legitimate
void M(object obj)
{
    if (obj is string) { }
}

// Nullable reference with variable — correct idiom
void M(string? str)
{
    if (str is string s)
    {
        Console.WriteLine(s);  // s is guaranteed non-null
    }
}

// NRT disabled — is Type may be an intentional null guard
#nullable disable
void M(string str)
{
    if (str is string) { }
}
```

## Code fix

**Use 'is not null'** — replaces `expr is Type` with `expr is not null`. Only offered when the expression is a nullable reference type (flow state is `MaybeNull`). Not offered for value types or non-nullable reference types where the check is always true.
