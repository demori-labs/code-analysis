# DL1001 - Records should not have mutable properties

Records are intended to be immutable data types. Properties with `set` accessors break this guarantee and should use `init` instead.

| Rule   | Category | Default Severity | Code Fix |
|--------|----------|-----------------|----------|
| DL1001 | Design   | Warning         | Yes      |

## Why

Records provide value-based equality and are designed around immutability. A `set` accessor allows mutation after construction, undermining the purpose of using a record in the first place. Using `init` ensures properties can only be assigned during object initialisation.

## Scope

This rule applies to:

- **Record classes** (`record` / `record class`)
- **Readonly record structs** (`readonly record struct`)

It does **not** apply to plain `record struct` declarations, which are mutable by design.

## Examples

### Violation

```csharp
public record Person
{
    public string Name { get; set; } // DL1001
    public int Age { get; set; } // DL1001
}
```

### Fixed

```csharp
public record Person
{
    public string Name { get; init; }
    public int Age { get; init; }
}
```

### Positional record with extra mutable property

```csharp
public record Person(string Name)
{
    public int Age { get; set; } // DL1001
}
```

Positional parameters themselves are not flagged (they are implicitly `init`-only).

### No diagnostic

```csharp
// Init-only properties
public record Person
{
    public string Name { get; init; }
}

// Get-only properties
public record Person
{
    public string Name { get; }
}

// Positional parameters
public record Person(string Name, int Age);

// Record struct — mutable by design, set is allowed
public record struct Point
{
    public int X { get; set; }
    public int Y { get; set; }
}
```

## Suppression

The `[Mutable]` attribute can be used to suppress this diagnostic, but it should be treated as a last resort for cases where mutability is genuinely required. Prefer restructuring the code to maintain immutability.

```csharp
using DemoriLabs.Diagnostics.Attributes;

// Suppress for the entire record
[Mutable]
public record Person
{
    public string Name { get; set; }
}

// Suppress for a single property
public record Person
{
    [Mutable]
    public string Name { get; set; }
    public int Age { get; init; }
}
```

## Configuration

```ini
# Adjust severity
dotnet_diagnostic.DL1001.severity = error

# Disable entirely
dotnet_diagnostic.DL1001.severity = none
```

## Code fix

**Replace 'set' with 'init'** — converts each `set` accessor to `init`, preserving all other formatting.
