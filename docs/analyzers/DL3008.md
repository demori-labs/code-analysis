# DL3008 - Simplify boolean return

Detects `if` statements that return `true`/`false` in both branches and suggests simplifying to a single `return` statement.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3008 | Style    | Warning          | Yes      |

## Why

An `if` statement that returns `true` in one branch and `false` in the other is equivalent to returning the condition itself (or its negation). Simplifying removes unnecessary control flow and makes the intent clearer.

## When the diagnostic fires

- `if (cond) return true; else return false;` -- with else clause
- `if (cond) return true; return false;` -- fall-through (no else, next statement is the opposite return)
- `if (cond) return false; else return true;` -- negated with else
- `if (cond) return false; return true;` -- negated fall-through
- Block-bodied variants of all of the above

## When the diagnostic does NOT fire

- Non-boolean return values (`if (cond) return 1; else return 0;`)
- Intermediate statements between `if` and the fall-through return
- No matching else or fall-through return
- Both branches return the same value (`return true`/`return true`)
- Multiple statements in the if body
- Inside `Expression<Func<>>` lambdas (expression trees)

## Examples

### Violation

```csharp
if (cond) return true; else return false;   // DL3008
if (cond) return false; return true;        // DL3008
if (a && b) { return true; } return false;  // DL3008
```

### Fixed

```csharp
return cond;
return cond is false;
return a && b;
```

### No diagnostic

```csharp
// Non-boolean
if (cond) return 1; else return 0;

// Intermediate statement
if (cond) return true;
DoSomething();
return false;

// Same value both branches
if (cond) return true; else return true;
```

## Code fix

When the `if` returns `true` and the else/fall-through returns `false`, replaces the entire construct with `return condition;`. When the `if` returns `false` and the else/fall-through returns `true`, replaces with `return condition is false;`. Preserves leading trivia from the original `if` statement.
