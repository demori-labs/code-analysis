# DL1003 - Record has too many positional parameters

Records with many positional parameters become hard to read at call sites and maintain over time. This rule suggests converting positional parameters to explicit `required` properties.

| Rule   | Category | Default Severity | Default Threshold |
|--------|----------|-----------------|-------------------|
| DL1003 | Design   | Warning         | 4                 |

The diagnostic is reported when the number of positional parameters **exceeds** the threshold (i.e. 5 or more with the default).

## Why

Positional parameters rely on argument order, which becomes error-prone as the parameter count grows:

```csharp
// Which string is which?
var person = new Person("Alice", "Smith", "alice@example.com", "07700900000", "123 Main St", "London");
```

Explicit required properties make construction self-documenting:

```csharp
var person = new Person
{
    FirstName = "Alice",
    LastName = "Smith",
    Email = "alice@example.com",
    Phone = "07700900000",
    Address = "123 Main St",
    City = "London",
};
```

## Examples

### Violation

```csharp
// DL1003: Record 'Person' has 6 positional parameters (threshold: 4)
public record Person(string FirstName, string LastName, int Age, string Email, string Phone, string Address);
```

### Fixed — explicit required properties

```csharp
public record Person
{
    public required string FirstName { get; init; }
    public required string LastName { get; init; }
    public required int Age { get; init; }
    public required string Email { get; init; }
    public required string Phone { get; init; }
    public required string Address { get; init; }
}
```

### Record struct — mutable properties

Record structs (without `readonly`) use `set` instead of `init`:

```csharp
// Before
public record struct Point(int X, int Y, int Z, int W, int U, int V);

// After
public record struct Point
{
    public required int X { get; set; }
    public required int Y { get; set; }
    public required int Z { get; set; }
    public required int W { get; set; }
    public required int U { get; set; }
    public required int V { get; set; }
}
```

### Readonly record struct — immutable properties

```csharp
// Before
public readonly record struct Point(int X, int Y, int Z, int W, int U, int V);

// After
public readonly record struct Point
{
    public required int X { get; init; }
    public required int Y { get; init; }
    public required int Z { get; init; }
    public required int W { get; init; }
    public required int U { get; init; }
    public required int V { get; init; }
}
```

### Optional parameters preserve defaults

Parameters with default values become non-`required` properties with initialisers:

```csharp
// Before
public record Person(string Name, int Age = 0, string? Country = "UK");

// After
public record Person
{
    public required string Name { get; init; }
    public int Age { get; init; } = 0;
    public string? Country { get; init; } = "UK";
}
```

### Constructor chaining

When the record has constructors that chain to the primary constructor via `: this(...)`, the code fix rewrites them to assign properties directly. Properties are not marked `required` in this case, since the constructors serve as the entry points:

```csharp
// Before
public record Person(string Name, int Age)
{
    public Person(string name) : this(name, 0) { }
}

// After
public record Person
{
    public string Name { get; init; }
    public int Age { get; init; }

    public Person(string name)
    {
        Name = name;
        Age = 0;
    }
}
```

Constructors that chain to other explicit constructors (not the primary) are preserved as-is:

```csharp
// Before
public record Person(string Name, int Age)
{
    public Person(string name) : this(name, 0) { }
    public Person() : this("Unknown") { }
}

// After
public record Person
{
    public string Name { get; init; }
    public int Age { get; init; }

    public Person(string name)
    {
        Name = name;
        Age = 0;
    }

    public Person() : this("Unknown") {}
}
```

## Configuration

### EditorConfig

```ini
# Change the threshold (default: 4, diagnostic fires when exceeded)
dotnet_diagnostic.DL1003.positional_parameters_threshold = 6

# Adjust severity
dotnet_diagnostic.DL1003.severity = warning
```

## Code fix

The code fix **Convert to explicit required properties** is available and handles:

- Converting positional parameters to `public required <type> <Name> { get; init; }` properties
- Using `get; set;` for mutable record structs, `get; init;` for everything else
- Preserving default values as property initialisers (without `required`)
- Rewriting constructors that chain to the primary constructor into property assignments
- Preserving constructors that chain to other explicit constructors
- Preserving existing members (computed properties, methods, etc.)
- Reading `indent_style` and `indent_size` from EditorConfig for correct formatting
