# DL1002 - Records should not have mutable property types

Record properties should use immutable or read-only collection types. Using mutable collections like `List<T>` allows the contents to be changed after construction, even if the property itself is `init`-only.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL1002 | Design   | Warning          | No       |

## Why

An `init`-only property holding a `List<T>` is still mutable — callers can `.Add()`, `.Remove()`, or `.Clear()` the collection freely. For records to be truly immutable, their properties must use types that prevent modification.

## Detected mutable types

### System.Collections.Generic

`List<T>`, `Dictionary<TKey, TValue>`, `HashSet<T>`, `SortedSet<T>`, `Queue<T>`, `Stack<T>`, `LinkedList<T>`, `SortedDictionary<TKey, TValue>`, `SortedList<TKey, TValue>`

### System.Collections

`ArrayList`, `Hashtable`

### System.Collections.ObjectModel

`Collection<T>`, `ObservableCollection<T>`

### System.Collections.Concurrent

`ConcurrentDictionary<TKey, TValue>`, `ConcurrentBag<T>`, `ConcurrentQueue<T>`, `ConcurrentStack<T>`

### Other

- Arrays (`T[]`, `int[]`, etc.)
- Custom types inheriting from any mutable type above (checked recursively through the type hierarchy)

## Examples

### Violation

```csharp
public record Person
{
    public List<string> Tags { get; init; } // DL1002
    public Dictionary<string, int> Scores { get; init; } // DL1002
    public int[] Values { get; init; } // DL1002
}
```

### Custom collection inheriting from mutable type

```csharp
public class MyCollection<T> : List<T> { }

public record Data
{
    public MyCollection<string> Items { get; init; } // DL1002
}
```

### Positional parameter with mutable type

```csharp
public record Data(List<string> Items); // DL1002
```

### No diagnostic

```csharp
public record Person
{
    public IReadOnlyList<string> Tags { get; init; }
    public ImmutableList<string> Items { get; init; }
    public FrozenSet<string> Roles { get; init; }
    public string Name { get; init; }
}
```

Regular classes and structs are not analysed — this rule applies only to record types.

## Suppression

The `[Mutable]` attribute can be used to suppress this diagnostic, but it should be treated as a last resort. Prefer switching to an immutable collection type instead.

```csharp
using DemoriLabs.CodeAnalysis.Attributes;

// Suppress for the entire record
[Mutable]
public record Person
{
    public List<string> Tags { get; init; }
}

// Suppress for a single property
public record Person
{
    [Mutable]
    public List<string> Tags { get; init; }
}
```

## Configuration

```ini
# Adjust severity
dotnet_diagnostic.DL1002.severity = error

# Disable entirely
dotnet_diagnostic.DL1002.severity = none
```

## Code fix

No automatic code fix. The correct immutable type depends on the use case (`IReadOnlyList<T>`, `ImmutableArray<T>`, `FrozenSet<T>`, etc.).
