# DL3006 - Use declaration pattern instead of type check and cast

Detects `if (x is T)` followed by a cast `(T)x` in the if-body, which should use a declaration pattern `if (x is T t)` instead.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3006 | Style    | Warning          | Yes      |

## Why

Using `is T` followed by a cast `(T)x` performs the type check twice — once for the `is` and once for the cast. A declaration pattern `is T t` combines both into a single operation, eliminating the redundant check and making the code more concise.

## When the diagnostic fires

- `if (x is T)` where the if-body contains one or more `(T)x` casts of the same expression to the same type
- The `is` expression is in the condition of an `if` statement (or part of a `&&` condition)

## When the diagnostic does NOT fire

- `x is T` used outside an if-condition (e.g. in a return expression)
- No matching casts found in the if-body
- Cast is to a different type than the type check
- Inside `Expression<Func<>>` lambdas
- Already using a declaration pattern `x is T t`

## Examples

### Violation

```csharp
if (o is Animal)              // DL3006
{
    var a = (Animal)o;
    Console.WriteLine(a.Name);
}
```

### Fixed

```csharp
if (o is Animal animal)
{
    var a = animal;
    Console.WriteLine(a.Name);
}
```

### No diagnostic

```csharp
// Already using declaration pattern
if (o is Animal animal)
{
    Console.WriteLine(animal.Name);
}

// No cast in the body
if (o is Animal)
{
    Console.WriteLine("it's an animal");
}
```

## Code fix

Replaces `x is T` with `x is T varName` in the if-condition, and replaces all matching `(T)x` casts in the if-body with `varName`. The variable name is derived from the type name in camelCase. C# keywords are prefixed with `@`.
