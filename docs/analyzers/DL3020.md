# DL3020 - File contains multiple different namespaces

Flags files that contain multiple different namespace declarations, preventing conversion to a file-scoped namespace.

| Rule   | Category | Default Severity | Code Fix |
| ------ | -------- | ---------------- | -------- |
| DL3020 | Style    | Warning          | No       |

## Why

A file should contain types from a single namespace. Multiple namespaces in one file violate the convention of matching namespace to folder structure (DL3018) and prevent the use of file-scoped namespace declarations (DL3019). They also make the file harder to navigate and reason about.

## When the diagnostic fires

- A file has multiple top-level block-scoped namespace declarations with different names
- A file has a nested namespace where the outer namespace also has its own direct members (types at different namespace levels)

## When the diagnostic does NOT fire

- File-scoped namespaces (the compiler already prevents duplicates)
- A single namespace declaration
- Multiple block namespaces with the same name (DL3019 handles this)
- Nested namespaces where the outer is a pure wrapper with no direct members (DL3019 handles this)

## Examples

### Violation

```csharp
namespace MyApp // DL3020
{
    public class A { }
}

namespace OtherApp // DL3020
{
    public class B { }
}
```

```csharp
namespace MyApp // DL3020
{
    public class A { }

    namespace Inner
    {
        public class B { }
    }
}
```

### No diagnostic

```csharp
// Single namespace
namespace MyApp;

public class A { }
```

```csharp
// Nested but outer is pure wrapper (DL3019 handles this)
namespace MyApp
{
    namespace Inner
    {
        public class A { }
    }
}
```

## Configuration

```ini
# Adjust severity
dotnet_diagnostic.DL3020.severity = error

# Disable entirely
dotnet_diagnostic.DL3020.severity = none
```
