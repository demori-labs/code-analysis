# DL5002 - Use string.IsNullOrEmpty

Flags empty string comparisons that should use `string.IsNullOrEmpty` instead.

| Rule   | Category    | Default Severity | Code Fix |
| ------ | ----------- | ---------------- | -------- |
| DL5002 | Performance | Warning          | Yes      |

## Why

`string.IsNullOrEmpty` is a JIT intrinsic that is faster than comparing against `""`, `string.Empty`, or checking `.Length == 0`. It also correctly handles null strings — unlike `str == ""` which silently returns false when the string is null, hiding a likely bug.

## When the diagnostic fires

- `str == ""` / `str != ""`
- `str == string.Empty` / `str != string.Empty`
- `str.Length == 0` / `str.Length != 0`
- `str.Length is 0` / `str.Length is not 0`
- `str?.Length == 0` / `str?.Length != 0`
- `str is ""` / `str is not ""`
- `str is null or ""`
- `str is not null and not ""`
- `string.Equals(str, "")` / `string.Equals(str, "", StringComparison.X)`
- `str.Equals("")` / `str.Equals(string.Empty)`

## When the diagnostic does NOT fire

- Non-empty string comparisons (`str == "hello"` — handled by DL3017)
- Null checks (`str == null` — handled by DL3003)
- Non-zero length checks (`str.Length == 5`)
- Non-string types (`array.Length == 0`)
- Inside expression trees
- Complex patterns (`str is null or "" or "N/A"`)

## Examples

### Violation

```csharp
public bool IsEmpty(string name)
{
    return name == ""; // DL5002
}

public bool HasValue(string? name)
{
    return name is not null and not ""; // DL5002
}

public bool CheckLength(string name)
{
    return name.Length == 0; // DL5002
}
```

### Fixed

```csharp
public bool IsEmpty(string name)
{
    return string.IsNullOrEmpty(name);
}

public bool HasValue(string? name)
{
    return string.IsNullOrEmpty(name) is false;
}

public bool CheckLength(string name)
{
    return string.IsNullOrEmpty(name);
}
```

### No diagnostic

```csharp
// Already using IsNullOrEmpty
if (string.IsNullOrEmpty(name)) { }

// Non-empty string (DL3017 handles this)
if (name == "admin") { }

// Non-string type
if (items.Length == 0) { }
```

## Code fix

Replaces the comparison with `string.IsNullOrEmpty(expr)`. For negated forms (`!=`, `is not`, `is not null and not ""`), the replacement is `string.IsNullOrEmpty(expr) is false`. For `.Length` comparisons, the receiver is extracted (e.g. `str.Length == 0` → `string.IsNullOrEmpty(str)`).

## Related Microsoft rules

DL5002 supersedes [CA1820](https://learn.microsoft.com/en-us/dotnet/fundamentals/code-analysis/quality-rules/ca1820) which only flags `str == ""` and `string.Equals(str, "")`. DL5002 additionally covers `.Length == 0`, pattern matching (`is ""`, `is null or ""`), and `string.Equals` with `StringComparison`. Disable CA1820 to avoid duplicate diagnostics:

```ini
dotnet_diagnostic.CA1820.severity = none
```
