# Contributing Guidelines

Thank you for your interest in contributing to this project! Before you start, please take a moment to review these guidelines:

 - [Questions and Enquiries](#questions-and-enquiries)
 - [Reporting Bugs](#reporting-bugs)
 - [Suggesting Features](#suggesting-features)
 - [Pull Requests](#pull-requests)
 - [Contributor Licence Agreement](#contributor-licence-agreement)

## Questions and Enquiries

If you have any questions regarding how to use this project, please feel free to open a discussion or an issue on the GitHub repository.

## Reporting Bugs

Should you discover a bug within the source code, please assist us by raising an issue. Even better, you are highly encouraged to submit a Pull Request containing a proposed fix.

When logging an issue, kindly provide the following details:

- A clear and descriptive summary of the problem.
- The exception message and full stack trace, if an error occurred.
- Where possible, code that reliably replicates the issue. You can use GitHub Gists or alternative pastebin services to share substantial code snippets. Alternatively, you could reproduce the bug in a new unit test within a pull request.

The more context you provide, the easier and quicker it will be to resolve the issue!

## Suggesting Features

If you have an idea for a new feature, you can request it by opening an issue on the GitHub repository. Please provide as much detail as possible about your proposed addition and its expected behaviour.

## Pull Requests

Prior to submitting a pull request, please ensure you have completed the following steps:

- Verify that any new or modified code adheres to the project's existing formatting and naming conventions.
- Execute the test suite to guarantee that no current functionality has been compromised.
- Include new unit tests for your changes. All newly introduced features and bug fixes must be accompanied by tests to validate they work correctly.

For further guidance on creating pull requests, please consult the GitHub Help documentation.
